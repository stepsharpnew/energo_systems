## Установка зависимостей

```bash
cd front
npm install
```

## Запуск проекта

```bash
npm run dev
```

Проект будет доступен по адресу: http://localhost:3000

## Основные изменения в коде

### Роутинг
- **Было**: Vue Router с явной конфигурацией
- **Стало**: File-based роутинг Nuxt
  - `pages/index.vue` → `/`
  - `pages/services/[id].vue` → `/services/:id`

### Импорты
- **Было**: `@/components/Component.vue`
- **Стало**: `~/components/Component.vue`

### Изображения
- **Было**: `require('@/assets/image.png')`
- **Стало**: `'/img/image.png'` (для статических файлов в `public/`)

### Состояние
- Vuex удален (был пустым)
- Можно использовать composables при необходимости

## Структура проекта

```
front/
├── app.vue                 # Главный компонент приложения
├── nuxt.config.ts          # Конфигурация Nuxt
├── pages/                  # Страницы (file-based routing)
│   ├── index.vue          # Главная страница
│   └── services/
│       └── [id].vue       # Страница услуги
├── components/             # Компоненты
│   └── energy-systems/     # Компоненты энергосистем
├── assets/                 # Ассеты (стили, данные)
│   ├── data/              # Данные
│   └── styles/            # Стили
├── plugins/                # Плагины
│   ├── vuetify.ts         # Плагин Vuetify
│   └── gsap.client.ts      # Плагин GSAP (только клиент)
└── public/                 # Статические файлы
    └── img/               # Изображения
```

## Важные замечания

1. **Vuetify**: Настроен и готов к использованию, но компоненты пока используют обычные HTML элементы
2. **GSAP**: Плагин настроен как `.client.ts` (только для клиента)
3. **SSR**: Nuxt поддерживает SSR, но GSAP работает только на клиенте
4. **Изображения**: Все изображения должны быть в `public/img/`
